# 《视传杀》完整游戏创作思路

## 1. 项目目标

《视传杀》定位为视觉传达专业主题的身份类策略卡牌游戏。整体体验高度参考经典身份局的结构：多人身份、武将技能、手牌管理、回合阶段、锦囊博弈、装备成长、阵营胜负。

为了避免直接复制现成商业素材，所有角色、牌面、桌面、令牌、按钮、面板都采用原创视觉方向，由 GPT 生成底图，再经过裁切、抠图、适配后嵌入网页。

## 2. 核心玩法规划

- 基础身份：主公、忠臣、反贼、内奸。
- 基础阶段：准备阶段、摸牌阶段、出牌阶段、弃牌阶段。
- 基础牌类：
  - 提案突袭：对应“杀”，造成 1 点压力。
  - 临场改稿：对应“闪”，抵消突袭。
  - 灵感补给：对应“桃”，回复 1 点体力。
  - 甲方需求：对应拆牌/干扰类锦囊。
  - 赶稿通宵：对应群体压力类锦囊。
  - 头脑风暴：对应摸牌类锦囊。
  - 数位板：对应装备牌，放宽突袭次数。
- 当前版本采用 1 名玩家 + 4 名 AI 的单机身份局。
- 后续目标扩展到 5、6、8 人可配置身份局。

## 3. 初始武将规划

- 马新宇：版式奇谋，技能【对齐】，偏过牌与整理。
- 曾宇鹏：镜头调度，技能【构图】，偏进攻收益。
- 陈豪杰：字体铸师，技能【字重】，偏进攻失败补偿。
- 曾敏俊：海报重装，技能【厚涂】，偏濒危保护。
- 李佳乐：动效游侠，技能【转场】，偏被攻击时的机动补偿。
- 任腾飞：策展先锋，技能【布展】，偏装备与攻击次数。
- 龚欣怡（女）：灵感缪斯，技能【共鸣】，偏回复与补牌。

周亦航已从当前角色池移除。

## 4. UI 设计规划

游戏内 UI 采用牌桌式布局：

- 上方：敌方武将席位，显示武将、体力、手牌数、隐藏身份令牌。
- 中央：牌桌区，显示阶段条、当前回合、牌堆、弃牌堆、目标选择。
- 下方：玩家主公区和手牌区。
- 左侧：模式规划与重开入口。
- 右侧：战局日志。

视觉风格方向：

- 古风木质牌桌、暗红漆面、黄铜边框、宣纸面板、墨色阴影。
- 角色是现代视觉传达专业武将，不做纯古代武将。
- UI 要有经典身份局的桌面感，但不复制任何现成游戏的图标、卡框、商标或界面。

## 5. 素材生产与抠图流程

当前资产管线：

1. 使用 GPT 生成角色绿幕图 `assets/cutouts/heroes-green-sheet.png`。
2. 使用 `tools/build_assets.py` 将角色按槽位裁切。
3. 通过颜色键抠除 #00ff00 绿幕。
4. 对 alpha 边缘进行轻微收缩与柔化。
5. 清理槽位边缘，避免残留相邻角色碎片。
6. 输出透明角色 PNG 到 `assets/processed/heroes/`。
7. 使用 GPT 生成 UI 素材图 `assets/ui/table-ui-sheet.png`。
8. 裁切出桌面背景、卡背、面板、阶段横幅、身份令牌等素材到 `assets/processed/ui/`。

后续建议：

- 为每个角色单独生成绿幕图，能得到更完整的身体边缘。
- 为每类牌生成独立卡面插画。当前基础牌、锦囊牌和数位板装备牌已完成第一版。
- 为按钮、身份牌、装备区、判定区生成可九宫格缩放的 UI 边框。

## 6. 技术实现规划

- 当前技术：纯 HTML、CSS、JavaScript，无构建工具，便于直接打开与迭代。
- 游戏状态集中在 `game.js` 的 `state`。
- 武将、牌堆、AI、胜负判断都在前端本地运行。
- 美术资产通过 `assets/processed/` 引用，避免直接依赖原始 GPT 大图。
- `tools/build_assets.py` 作为可重复运行的资产构建脚本。

## 7. 下一阶段优先级

1. 增加手动响应窗口：玩家被突袭时选择是否打出【临场改稿】。
2. 增加濒危求桃：自己或队友濒危时可响应【灵感补给】。
3. 增加距离与装备区：武器、防具、坐骑、数位板细化。
4. 增加判定区与延时锦囊。
5. AI 改成隐藏身份推理，而不是完全明牌决策。
6. 为牌面生成独立插画并做卡面适配。
7. 加入开始动画、出牌动画、伤害动画和阵亡翻身份动画。
